# 次のスレッド用プロンプト（コピーして新規チャットに貼り付けてください）

---

このリポジトリは Next.js のポートフォリオサイト（frontend/ がメイン）です。前スレッドで「保守性・可読性・不要ファイルの整理」を一括で実施済みです。

**次のスレッドで詰めてほしいこと：**

1. **Storybook のカバー拡大**
   - 現状: `src/stories/Button.stories.tsx` のみ（`@/components/ui/Button` を参照）。
   - 対応: 実際に使っている UI コンポーネント（`Section`, `SectionHeading`、必要なら `ContactForm` など）のストーリーを追加し、Storybook で仕様・見た目を確認できるようにする。

2. **E2E テストの見直し**
   - 現状: Cypress で `contact.cy.ts` など一部のみ。
   - 対応: 重要な画面・フロー（トップ、お問い合わせ、/products/picsche など）の E2E カバーを整理・拡張する。テストの目的と範囲を明確にする。

3. **URL・パス表記の明文化**
   - 現状: 正規はすべて小文字（例: `/products/picsche`）。大文字（`PicSche`）は macOS の case-insensitive で同じディレクトリを指すことがあり、IDE で `PicSche` と表示されることがある。
   - 対応: ルート README または `frontend/docs/README.md` に「リポジトリ内のパス・URL はすべて小文字で統一している」旨を 1 行で追記し、将来の混乱を防ぐ。

4. **その他、品質・一貫性の確認**
   - 上記のほか、パフォーマンス・アクセシビリティ・エラーハンドリングなど、サイトの品質を一段上げるために必要な作業があれば提案し、優先度をつけて対応する。

**前提・制約（ユーザールールに従うこと）：**
- 会話は日本語。
- 修正後はビルドが通ること。ビルドエラーが出たら解消するまで対応する。
- 自動テストは「テスト結果をいじってパスさせる」のではなく、仕様と実装が正しいかを検証する。
- `.env` / `.env.local` は削除・上書きしない（.cursor/rules/env-files.mdc 参照）。
